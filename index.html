    2 裝上新的硬碟

    過程如下：

    • [X] setup HD: connect to computer
    • [X] gparted: create partition and format it in NTFS. 結果是在 windows 10 中做 partition 和 format.
    • [X] SMART check both old and new HD
    • [X] mount to linux
    • [X] sync from old to new HD: rsync -av <src> <desc>
    • [X] if old HD is ok, then add it to lvm otherwise drop it
    • [X] disconnet old HD, reboot to windows 10
    • [X] set new HD to disk D:

    要加到 lvm 就看 replace hd on LVM 這篇記錄，有經驗，不會太難。

    2.3.1 刪除不必要的空白

    換行時，在兩個中文字間會有一個空白，不好看，所以想辦法刪除。 

    ;; https://coldnew.github.io/a1ed40e3/
(defadvice org-html-paragraph (before org-html-paragraph-advice
				      (paragraph contents info) activate)
  "Join consecutive Chinese lines into a single long line without
unwanted space when exporting org-mode to html."
  (let* ((origin-contents (ad-get-arg 1))
	 (fix-regexp "[[:multibyte:]]")
	 (fixed-contents
	  (replace-regexp-in-string
	   (concat
	    "\\(" fix-regexp "\\) *\n *\\(" fix-regexp "\\)") "\\1\\2" origin-contents)))

    (ad-set-arg 1 fixed-contents)))

    2.3.2 Convert inline verbatim to kbd when HTML export

    ref: https://emacs-china.org/t/org-mode/1165

    change verbatim to kbd

    code 

    (setq org-html-text-markup-alist
      '((bold . "<b>%s</b>")
	(code . "<code>%s</code>")
	(italic . "<i>%s</i>")
	(strike-through . "<del>%s</del>")
	(underline . "<span class=\"underline\">%s</span>")
	(verbatim . "<kbd>%s</kbd>")))

    2.3.3 用 pygment 幫程式碼著色

    $ cd themes/your-theme-name/assets/css
$ pygmentize -S emacs -a .highlight -f html > pygment.css

    2.4 用 nikola github_deploy 快速發佈文章

    REF: https://getnikola.com/handbook.html#deploying-to-github

    2.4.1 Initialize a Nikola site, if you haven’t already.

    2.4.2 Initialize a git repository in your Nikola source directory by running:

    git init .
git remote add origin git@github.com:user/user.github.io

    url 要確認清楚，必須是 ssh 形式的，未來才可不必一直重覆輸入密碼

    2.4.3 Setup branches and remotes in conf.py:

    • GITHUB_DEPLOY_BRANCH is the branch where Nikola-generated HTML files will be deployed. It should be gh-pages for project pages and master for user pages (user.github.io).
    • GITHUB_SOURCE_BRANCH is the branch where your Nikola site source will be deployed. We recommend and default to src.
    • GITHUB_REMOTE_NAME is the remote to which changes are pushed.
    • GITHUB_COMMIT_SOURCE controls whether or not the source branch is automatically committed to and pushed. We recommend setting it to True, unless you are automating builds with Travis CI.

    2.4.4 Create a .gitignore file. We recommend adding at least the following entries:

    cache
.doit.db
__pycache__
output

    If you set GITHUB_COMMIT_SOURCE to False, you must switch to your source branch and commit to it. Otherwise, this is done for you.

    2.4.5 Run nikola github_deploy.

    This will build the site, commit the output folder to your deploy branch, and push to GitHub. Your website should be up and running within a few minutes.

    2.4.6 在 nikola github_deploy 之前

    為了要能用 nikola github_deploy 指令，有以下的準備工作：

    1. install ghp_import2
      sudo pip install ghp_import2
    2. install and setup ssh

      為了不必一直重覆輸入 github 的帳密，可用 ssh 解決。

      1. install openssh
        yaourt -S openssh
      2. follow github help page to add ssh connect to github
        • Generating a new SSH key and adding it to the ssh-agent - User Documentation
        • Working with SSH key passphrases - User Documentation
        • Adding a new SSH key to your GitHub account - User Documentation
        1. 產生 ssh key
          ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

          螢幕上會出現 

          Enter passphrase for key '/home/USERNAME/.ssh/id_rsa'

          此時不要輸入密碼，直接按 Enter ，否則未來 nikola github_deploy 時，就要每次輸入密碼。

        2. 將 ssh key 加到 github 中

          依 Adding a new SSH key to your GitHub account - User Documentation 執行

    3. 確定以 ssh 連上 github (不能是 https)

    音響要好聲，空間才是王道1，没有 DRC 前，先試試等化器2，也許調一調就可用啦。在 archlinuxl 中，我用的是 alsa 的架構播放音樂， Archlinux WIKI 的 Advanced linux Sound Architecture 已說明等化器的安裝及設定，照做即可。

    1 install

    yaourt -S alsaequal

    2 config

    在 /etc/asound.conf 加上以下設定： 

    ctl.equal {
    type equal;
}

pcm.plugequal {
    type equal;
    # Modify the line below if you do not
    # want to use sound card 0.
    #slave.pcm "plughw:0,0";
    # by default we want to play from more sources at time:
    slave.pcm "plug:dmix";
}

# pcm.equal {
# If you do not want the equalizer to be your
# default soundcard comment the following
# line and uncomment the above line. (You can
# choose it as the output device by addressing
# it with specific apps,eg mpg123 -a equal 06.Back_In_Black.mp3)
pcm.!default {
    type plug;
    slave.pcm plugequal;
}

    3 reboot

    讓 alsa 可用等化器

    4 調整等化器

    執行 

    alsamixer -D equal

    PS. 用 Q,W,E,Z,X,C 可分別調整左右聲道的音量。

    Note that configuration file is different for each user (until not specified else) it is saved in ~/.alsaequal.bin. so if you want to use ALSAEqual with mpd or another software running under different user, you can configure it using

    or for example, you can make a symlink to your .alsaequal.bin in his home…
