    掙扎了許久，還是先用 nikola 記錄一些事，畢竟比較熟悉，能快一點上手。

    1 安裝

    sudo pip install nikola webassets ghp_import

    其中 ghp_import 是為了發佈到 github. webassets 可將 所有的 css 檔打包成一個 css 檔。

    2 設定

    執行 nikola init 後，依指示設定，比較省事。 

    nikola init <blog-name>
cd <blog-name>

    2.1 configuration file

    修改 conf.py ，慢慢改，內容很多，不必急於一時。

    2.2 theme

    nikola theme -l
nikola theme install <theme-name>

    2.3 用 nikola github_deploy 快速發佈文章

    參考 The Nikola Handbook | Nikola

    設定的步驟如下：

    1. Initialize a Nikola site, if you haven’t already.

    2. Initialize a git repository in your Nikola source directory by running: 

         git init .
   git remote add origin git@github.com:user/user.github.io

      url 要確認清楚，必須是 ssh 形式的，未來才可不必一直重覆輸入密碼

    3. Setup branches and remotes in conf.py:
      • GITHUB_DEPLOY_BRANCH is the branch where Nikola-generated HTML files will be deployed. It should be gh-pages for project pages and master for user pages (user.github.io).
      • GITHUB_SOURCE_BRANCH is the branch where your Nikola site source will be deployed. We recommend and default to src.
      • GITHUB_REMOTE_NAME is the remote to which changes are pushed.
      • GITHUB_COMMIT_SOURCE controls whether or not the source branch is automatically committed to and pushed. We recommend setting it to True, unless you are automating builds with Travis CI.

    4. Create a .gitignore file. We recommend adding at least the following entries: 

         cache
   .doit.db
   __pycache__
   output

      If you set GITHUB_COMMIT_SOURCE to False, you must switch to your source branch and commit to it. Otherwise, this is done for you.

    5. Run nikola github_deploy. This will build the site, commit the output folder to your deploy branch, and push to GitHub. Your website should be up and running within a few minutes.

    2.3.1 在 nikola github_deploy 之前

    為了要能用 nikola github_deploy 指令，有以下的準備工作：

    1. install ghp_import2
      sudo pip install ghp_import2
    2. install and setup ssh

      為了不必一直重覆輸入 github 的帳密，可用 ssh 解決。

      1. install openssh
        yaourt -S openssh
      2. follow github help page to add ssh connect to github

        參考了以下文章設定 SSH：

        • Generating a new SSH key and adding it to the ssh-agent - User Documentation
        • Working with SSH key passphrases - User Documentation
        • Adding a new SSH key to your GitHub account - User Documentation

        a. 產生 ssh key 

           ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

        螢幕上會出現 

           Enter passphrase for key '/home/USERNAME/.ssh/id_rsa'

        此時不要輸入密碼，直接按 Enter ，否則未來 nikola github_deploy 時，就要每次輸入密碼。

        b. 將 ssh key 加到 github 中依 Adding a new SSH key to your GitHub account - User Documentation 執行

    3. 確定以 ssh 連上 github (不能是 https 型式)
      $ git remote -v
origin  git@github.com:mariolong/mariolong.github.io (fetch)
origin  git@github.com:mariolong/mariolong.github.io (push)

      如果不是 ssh 型式，要用以下指令修正。 

      git remote set-url origin git@github.com:mariolong/mariolong.github.io

    3 用 org-mode 寫文章

    先安裝 plugin orgmode，再做一點點微調即可。 

    nikola plugin -i orgmode

    3.1 修改 /orgmode/init.el

    開啟 <blog-name>/plugins/orgmode/init.el ，加上以下的程式碼。

    3.1.1 刪除不必要的空白

    換行時，在兩個中文字間會有一個空白，不好看，所以想辦法刪除。 

    ;; https://coldnew.github.io/a1ed40e3/
(defadvice org-html-paragraph (before org-html-paragraph-advice
				      (paragraph contents info) activate)
  "Join consecutive Chinese lines into a single long line without
unwanted space when exporting org-mode to html."
  (let* ((origin-contents (ad-get-arg 1))
	 (fix-regexp "[[:multibyte:]]")
	 (fixed-contents
	  (replace-regexp-in-string
	   (concat
	    "\\(" fix-regexp "\\) *\n *\\(" fix-regexp "\\)") "\\1\\2" origin-contents)))

    (ad-set-arg 1 fixed-contents)))

    3.1.2 Convert inline verbatim to kbd when HTML export

    ref: https://emacs-china.org/t/org-mode/1165

    change verbatim to kbd

    code 

    (setq org-html-text-markup-alist
      '((bold . "<b>%s</b>")
	(code . "<code>%s</code>")
	(italic . "<i>%s</i>")
	(strike-through . "<del>%s</del>")
	(underline . "<span class=\"underline\">%s</span>")
	(verbatim . "<kbd>%s</kbd>")))

    3.1.3 在 Inline verbatim and code 不要空白

    如果有中文字要強調時，原本是要加上空白，如下： 

    ~hello~ hh
=hello= hh
*star* s
+中文+ 測試
/中文/ 測試
    • hello hh
    • hello hh
    • star s
    • 中文 測試
    • 中文 測試

    實在是不符習慣。google 了許久，加上一些實驗，終於找到以下的解法：

    開啟 <blog-name>/plugins/orgmode/init.el ，加上以下的程式碼。 

    (setq my-full-punctuation "。，：！？；、（）「」【】《》")
(setcar org-emphasis-regexp-components (concat " \t('\"{[:alpha:]" my-full-punctuation))
(setcar (nthcdr 1 org-emphasis-regexp-components) (concat "[:alpha:]- \t.,:!?;'\")}\\" my-full-punctuation))
(org-set-emph-re 'org-emphasis-regexp-components org-emphasis-regexp-components)

    結果如下： 

    - 測試~code~測試
- 測試=kbd=測試
- 測試*star*測試
- *star*s
- ~中文~測試
- =中文=測試
- *中文*測試
- +中文+測試
- /中文/測試

    3.1.4 用 pygment 幫程式碼著色

    1. 如果不想以 pygments 高亮程式碼

      在 init.el 設定 nikola-use-pygments 為 nil. 

      (setq nikola-use-pygments nil)
    2. 產生 pygments.css
      $ cd themes/your-theme-name/assets/css
$ pygmentize -S default -a .highlight -f html > pygments.css
    3. 設定 pygments 可支援的語言

      我目前加上 ini, systemd, shell. 可參考 Babel: Languages 和 Available lexers — Pygments 增減。 

      (defconst org-pygments-language-alist
  '(("asymptote" . "asymptote")
    ("conf" . "ini")
    ("ini" . "ini")
    ("systemd" . "sv")
    ("shell" . "sh")
    ("tcl" . "tcl"))
  "Alist between org-babel languages and Pygments lexers.
lang is downcased before assoc, so use lowercase to describe language available.
See: http://orgmode.org/worg/org-contrib/babel/languages.html and
http://pygments.org/docs/lexers/ for adding new languages to the mapping.")
    4. 程式碼縮排

      如果要保留整體縮排，加上： 

      (setq org-src-preserve-indentation t)

      不過，我覺得預設的 nil 可能好看點。

    5. 如果不想用 pygments

      其實我根本不想用 pygments，只想用內定的高亮型式。於是找到 init.el 把 org-html-src-block 函數刪掉。 

      ;; Override the html export function to use pygments
;; (defun org-html-src-block (src-block contents info)

      並加上 

      (setq nikola-use-pygments nil
      org-html-htmlize-output-type 'css)

    3.2 揷入圖片

    文章中不免有圖片，依照目前的工作流程，還是直接用 google photos 中的圖片，會比較方便。

    3.2.1 利用 Google Drive 取得 Google Photos 的照片

    參考 How to embed image from Google Drive in a Web page 這篇文章的方法。

    新的設定步驟如下：

    1. 在 google drive 中設定「建立 Google 相簿資料夾」，順便看一下

    Google 相簿如何與 Google 雲端硬碟搭配運作 - Google 雲端硬碟說明

    1. 在 google photots 中 設定「Google 雲端硬碟：從 Google 雲端硬碟同步處理相片和影片」

    2. 由 google drive 中選擇要分享的照片，按「右鍵」，選「取得檔案共用連結」，在剪貼簿中可得到下面連結： 

      https://drive.google.com/open?id=1Ni5UkMIN5AcTNYj10YS5OcK6UnMCFlHm

      註：這一步要善用 google drive 的「搜尋」功能。

    3. 把上述的 open? 改成 uc?export=view& ，並加上 <img> tags 即可。

    由以上 4 步驟，可得到以下指令： 

    #+HTML: <img src='https://drive.google.com/uc?export=view&=id=1Ni5UkMIN5AcTNYj10YS5OcK6UnMCFlHm'  width=320 alt="test picture"/>

    3.5 Subscripts and superscripts

    a_subb^sup

    如果真的要上、下標，就加 {} 

    a_{sub}b^{sup}

    結果為 asubbsup

    4 開始使用

    建立新的文章， 

    nikola new-post -e -f orgmod

    寫完就 build 

    nikola build

    或發佈吧！ 

    nikola github_deploy
